con github cli puedes ver los issues que estan en completed en un project?

¡Sí, totalmente! Con la CLI de GitHub (gh) puedes consultar el estado de los items dentro de un GitHub Project (Beta/v2), pero la forma de hacerlo es un poco distinta a como buscas issues normales, ya que los proyectos usan campos personalizados (como Status) para definir qué está "Completed".

Aquí tienes el comando exacto y los pasos para lograrlo:

El comando principal
Para ver los elementos de un proyecto, se utiliza el subcomando gh project item-list. El truco para ver solo los completados es usar el flag --format=json junto con jq (una herramienta para filtrar JSON) para quedarte solo con lo que esté en la columna de terminados.

Bash
gh project item-list <número-del-proyecto> --owner "<usuario-o-org>" --format json | jq '.items[] | select(.status == "Done") | {title: .content.title, number: .content.number, type: .content.type}'
💡 Guía paso a paso
Si no estás seguro de los datos que necesitas poner en el comando, sigue estos pasos:

Busca el número de tu proyecto:
Si no te lo sabes de memoria, lístalos con:

Bash
gh project list --owner "tu-usuario-o-org"
Esto te dará el número (ID corto) del proyecto.

Asegúrate de cómo se llama tu columna de completados:
Por defecto suele llamarse "Done" o "Closed", pero si la renombraste a "Completed", tendrás que cambiar ese texto en el filtro de jq:

Bash
# Cambia "Done" por "Completed" si es tu caso
... | jq '.items[] | select(.status == "Completed")'
🎯 Una alternativa más simple (si están cerrados)
Si en tu flujo de trabajo los issues "Completed" también están cerrados (closed) en el repositorio, a veces es más rápido listarlos directamente desde el repo filtrando por su estado, sin meterte con el comando de proyectos:

Bash
gh issue list --state closed
